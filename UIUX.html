1. Introduction

UI (User Interface): The visual elements of a product (buttons, icons, typography, color, spacing, layouts).

UX (User Experience): The overall experience users have when interacting with a product (ease of use, emotions, satisfaction).

Key Difference: UI = how it looks, UX = how it works & feels.

2. Design Basics

Principles of Design

  • Contrast ‚Üí makes important elements stand out.
  • Alignment ‚Üí everything should be visually connected.
  • Repetition ‚Üí reuse styles/components for consistency.
  • Proximity ‚Üí group related items together.

Typography

  • Use a maximum of 2‚Äì3 fonts in a project.
  • Maintain a clear hierarchy: H1 > H2 > body text.
  • Ensure readability (14‚Äì16px body text minimum).

Color Theory

  • Blue = trust (finance, healthcare apps).
  • Red = urgency, warning (errors, notifications).
  • Green = success, growth.
  • Use tools like Adobe Color or Coolors for palettes.

Grid Systems

  • Follow the 8px rule (margins, paddings, spacing in multiples of 8).
  • Common layouts: 12‚Äëcolumn grids for web, 4/8 grids for mobile.

4. UI Design Skills

  • Components: Reusable buttons, forms, cards, navbars.
  • Imagery & Icons: Use SVG icons for scalability.
  • Responsive Design: Test on mobile, tablet, desktop.
  • Accessibility: Maintain color contrast, add alt text, and enable keyboard navigation.
