1. Introduction

UI (User Interface): The visual elements of a product (buttons, icons, typography, color, spacing, layouts).

UX (User Experience): The overall experience users have when interacting with a product (ease of use, emotions, satisfaction).

Key Difference: UI = how it looks, UX = how it works & feels.

2. Design Basics

Principles of Design

  • Contrast → makes important elements stand out.
  • Alignment → everything should be visually connected.
  • Repetition → reuse styles/components for consistency.
  • Proximity → group related items together.

Typography

  • Use a maximum of 2–3 fonts in a project.
  • Maintain a clear hierarchy: H1 > H2 > body text.
  • Ensure readability (14–16px body text minimum).

Color Theory

  • Blue = trust (finance, healthcare apps).
  • Red = urgency, warning (errors, notifications).
  • Green = success, growth.
  • Use tools like Adobe Color or Coolors for palettes.

Grid Systems

  • Follow the 8px rule (margins, paddings, spacing in multiples of 8).
  • Common layouts: 12‑column grids for web, 4/8 grids for mobile.

3. UX Fundamentals

User Research

  • Conduct interviews, surveys, field studies.
  • Identify pain points, goals, and user behavior.

Personas

Create fictional characters representing user types.
Example: Ravi, 24, Student. Needs a budgeting app with simple navigation.

User Flows

Map how a user navigates:
Example: Login → Dashboard → Add Expense → Save → Review Report

Wireframing

  • Start with low‑fidelity sketches on paper or whiteboard.
  • Move to digital wireframes in Figma or Balsamiq.

Prototyping

  • Create clickable mockups with Figma, Adobe XD, or InVision.
  • Test flows before development.

4. UI Design Skills

  • Components: Reusable buttons, forms, cards, navbars.
  • Imagery & Icons: Use SVG icons for scalability.
  • Responsive Design: Test on mobile, tablet, desktop.
  • Accessibility: Maintain color contrast, add alt text, and enable keyboard navigation.

5. Design Tools

  • Figma → Most popular, free, web‑based. Great for collaboration.
  • Adobe XD → Good prototyping tool (part of Adobe suite).
  • Sketch → Popular among macOS designers.
  • Miro → Brainstorming & user journey mapping.
  • InVision → Prototyping & collaboration.

6. Practical Projects

  1. Redesign a Login Page
    Create a clean, minimal login/signup form in Figma. Include Google/Apple login options.

  2. E‑Commerce Landing Page
    Hero section with product image & CTA (Call to Action). Product grid, filters, and checkout flow.

  3. Mobile App Prototype
    Example: Expense tracker app. Screens: Login → Dashboard → Add Expense → Charts. Make it clickable in Figma.

💡 Pro Tip: Upload your designs to Dribbble, Behance, or LinkedIn to get feedback.

7. Usability Testing

  • A/B Testing → Compare two design versions.
  • Heatmaps → Use Hotjar to see where users click.
  • User Feedback → Ask real users to test prototypes.
  • Iteration → Improve designs based on test results.

8. Portfolio & Career

  • Portfolio: Show 3–5 strong projects.
  • Case Studies: Present problem → research → design process → final solution.
  • Job Prep: Common interview question: “Walk me through your design process.”
  • Freelancing: Platforms like Upwork, Fiverr, Toptal.
